# Screenshots for Landing Page V2

This directory contains screenshots used on the landing page to showcase the application features.

## Required Screenshots

Capture these screenshots manually from a live session with real/demo data:

| Filename | View | Description |
|----------|------|-------------|
| `landing-dashboard.png` | `/app/dashboard` | Full dashboard with stats, weekly progress, analytics |
| `landing-quick-capture.png` | Quick Capture Modal | Modal opened via Ctrl+K with empty form |
| `landing-application-detail.png` | `/app/applications/:id` | Application header with company, role, status badge |
| `landing-timeline.png` | Timeline Section | Timeline events showing progression |
| `landing-analytics.png` | Analytics Section | "What's Working" panel with keywords + conversion rates |
| `landing-charts.png` | Chart Section | Pie charts by status and source |
| `landing-contacts.png` | Contacts Section | Contact cards in application detail |
| `landing-cover-letter.png` | Cover Letter Panel | Cover letter generation/display |

## Specifications

- **Resolution**: 1440x900 minimum (desktop view)
- **Format**: PNG (universal compatibility)
- **Capture Method**: Manual capture from live session with real data
- **State**: Populated with realistic demo data

## Capture Instructions

1. Seed demo data in local/staging environment
2. Navigate to each view listed above
3. Capture full-width screenshots at desktop resolution
4. Save with exact filenames listed above
5. Optionally compress with tools like TinyPNG for faster loading

## Notes

The landing page includes fallback placeholder styles if screenshots are missing.
Screenshots should be updated when major UI changes occur.
