(1) RESUME_TEXT (the candidate’s resume)

GOAL
Generate a one-page, professional cover letter in clean, print-ready HTML that can be converted to a PDF. The tone should sound human—confident, direct, and tailored—without being overly long or “template-y.”

OUTPUT REQUIREMENTS (HARD)
- Output MUST be valid HTML only (no markdown, no backticks, no commentary).
- Single page when rendered to US Letter (8.5x11) with 1in margins.
- Use simple, PDF-friendly HTML + inline CSS in a <style> block.
- Use a single wrapper <div class="page"> with a max content width suitable for printing.
- Use fonts that render well everywhere (system fonts).
- No external assets, no links required (only include URLs if present in the resume).
- No tables unless needed for alignment; prefer simple <div>/<p> structure.
- 3–4 short paragraphs in the letter body (plus header + closing).
- Target length: ~250–400 words. Absolute max: 420 words.
- No bullet lists (ATS-safe and clean print), unless the job explicitly asks for them (rare).

TRUTHFULNESS + EVIDENCE (HARD)
- Every accomplishment, metric, tool, title, employer, certification, and claim must be supported by RESUME_TEXT.
- Company/role details (company name, role title) must come from JOB_DESCRIPTION_TEXT.
- If the job asks for a skill not in RESUME_TEXT, do NOT claim it. You may bridge honestly using adjacent experience (e.g., “building on my experience in X, I’m excited to grow into Y”) only if X exists in RESUME_TEXT.
- Avoid empty adjectives and clichés (e.g., “hard-working,” “team player,” “perfect fit,” “go-getter”). Prefer concrete proof.

CONTENT STRUCTURE (BEST PRACTICE)
1) Header block (top of page)
   - Candidate name (only if clearly present in RESUME_TEXT)
   - Email / phone / location / LinkedIn / portfolio (include only what’s present in RESUME_TEXT)
2) Letter body (3–4 short paragraphs)
   Paragraph 1: Role + company + a crisp thesis of fit (1 sentence hook + 1 sentence thesis).
   Paragraph 2: 1–2 highly relevant achievements mapped to the top job requirements (use metrics if available).
   Paragraph 3: Additional fit (collaboration, scope, tools, domain), and why this company/role (based ONLY on JOB_DESCRIPTION_TEXT).
   Paragraph 4: Close with interest in an interview and a polite thank-you.
3) Sign-off
   - “Sincerely,” then candidate name (only if name is clearly present in RESUME_TEXT)

STYLE GUIDELINES
- Human, concise, and specific.
- Vary sentence length and openings; don’t start every sentence with “I.”
- Use natural professional collocations (“proven track record,” “delivered,” “led,” “implemented,” “improved,” “reduced,” etc.).
- Use 1–2 transitions (“For example,” “As a result,” “In doing so,”) for flow.
- Keep it skimmable: short paragraphs, no walls of text.

PRINT/PDF HTML TEMPLATE REQUIREMENTS
- Include a <style> block that sets:
  - @page size: letter; margin ~1in (or equivalent)
  - body font: 11pt; line-height: 1.25–1.35
  - consistent spacing between header and paragraphs
- Use simple classes: .page, .header, .name, .contact, .date (optional), .salutation, .body, .closing
- Do NOT include the current date unless it appears in RESUME_TEXT or JOB_DESCRIPTION_TEXT. (If neither provides a date, omit.)

D) VERIFICATION PASS:

   - Confirm word count within limits.

E) Output ONLY the final HTML.
